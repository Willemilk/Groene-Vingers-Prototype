# Roadmap: Groene Vingers

**Created:** 2026-02-03
**Milestone:** v1.0 - AI Gardening Assistant

## Overview

7 phases to build full-featured gardening advice web application with Claude AI chatbot, user accounts, garden profiles, and conversation history.

## Phases

### Phase 1: Project Setup & Branch Creation
**Goal:** Set up development branch and project structure
**Requirements:** DEPL-01
**Success Criteria:**
1. New Git branch created (e.g., `feature/full-app`)
2. Project structure established (frontend, backend, database)
3. Development environment configured
4. Basic Next.js app running locally

---

### Phase 2: Authentication System
**Goal:** Users can create accounts and log in
**Requirements:** AUTH-01, AUTH-02, AUTH-03, AUTH-04
**Success Criteria:**
1. User can sign up with email/password
2. User can log in and see authenticated state
3. Session persists across browser refresh
4. User can log out successfully
5. Auth state managed globally

---

### Phase 3: Chat Interface & Claude Integration
**Goal:** Users can chat with Claude AI for gardening advice
**Requirements:** CHAT-01, CHAT-02, CHAT-03, CHAT-04, CHAT-05
**Success Criteria:**
1. Chat page accessible from navigation
2. User can type and send messages
3. Messages sent to Claude API via v0.app ai-tools
4. AI responses displayed in chat
5. Multilingual responses work
6. Real-time conversation flow feels natural

---

### Phase 4: Garden Profile System
**Goal:** Users can save garden information for personalized advice
**Requirements:** PROF-01, PROF-02, PROF-03, PROF-04, PROF-05
**Success Criteria:**
1. User can create garden profile with location
2. User can add plants to profile
3. User can add garden conditions
4. Profile data saved to database
5. Claude receives profile context with queries
6. Advice is personalized based on profile

---

### Phase 5: Conversation History
**Goal:** Users can save and review past conversations
**Requirements:** HIST-01, HIST-02, HIST-03, HIST-04
**Success Criteria:**
1. Conversations automatically saved during chat
2. User can see list of past conversations
3. User can open and read past conversations
4. User can continue previous conversations
5. User can delete unwanted conversations

---

### Phase 6: Content Pages
**Goal:** Complete the website with Services, About, and Contact pages
**Requirements:** PAGE-01, PAGE-02, PAGE-03, PAGE-04, PAGE-05
**Success Criteria:**
1. Diensten page with service information
2. Over ons page with background
3. Contact page with working contact form
4. All pages in Dutch
5. All pages match glassmorphism design
6. Navigation works across all pages

---

### Phase 7: Deployment & Polish
**Goal:** Deploy to Vercel and ensure production-ready
**Requirements:** DEPL-02, DEPL-03, DEPL-04
**Success Criteria:**
1. Environment variables configured in Vercel
2. Branch deployed to Vercel
3. Production build works without errors
4. Responsive design tested on mobile/desktop
5. All features working in production

---
